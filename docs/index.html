📖 ¿Qué es este proyecto?

Este proyecto es un generador automático de orlas académicas interactivas para centros educativos. A partir de una plantilla central y un conjunto de datos (profesorado, alumnado, fotografías), genera páginas HTML autónomas que pueden visualizarse en cualquier navegador o imprimirse directamente.

Cada orla incluye profesorado y alumnado con fotografía, nombre, cargo y un diploma personalizado con superpoderes aleatorios. Todo ello con un diseño clásico de orla escolar, sin necesidad de servidor, base de datos ni conexión a internet (excepto para las tipografías).

⚙️ Características detalladas

🖱️ Interactividad total

Cada persona en la orla es un elemento interactivo. Al hacer clic se abre un modal con su foto ampliada, nombre completo y cargo (si es profesor). Además, se genera un superpoder aleatorio para esa persona, ideal para hacer la entrega de orlas más amena y divertida.

▶️ Modo Recorrido automático

Activa el modo recorrido y la orla irá mostrando cada persona una a una automáticamente. Incluye:

  • Barra de progreso visual en la parte inferior
  • Intervalo de tiempo configurable entre persona y persona
  • Pausa / reanudación en cualquier momento
  • Confeti al completar el recorrido como celebración

🔍 Buscador instantáneo

Escribe en el campo de búsqueda (atajo: tecla F) y filtra al instante tanto profesorado como alumnado. Ideal para orlas con muchos estudiantes donde encontrar a una persona rápidamente.

📜 Diploma personalizado

Cada persona puede recibir un diploma único con:

  • Su nombre completo
  • Un superpoder generado aleatoriamente de una lista de más de 30
  • Diseño elegante con marco decorativo y tipografía especial
  • Listo para imprimir directamente desde el modal

🌓 Modo oscuro / claro

Alterna entre el tema oscuro (por defecto, ideal para proyecciones) y el tema claro (ideal para impresión). La preferencia se guarda en localStorage y persiste aunque recargues la página.

🖨️ Impresión optimizada

Pulsa el botón de imprimir y el navegador generará un PDF en horizontal con solo la orla visible. Mediante @media print se ocultan automáticamente todos los elementos de interfaz: botones, modales, buscador, barra de progreso, diploma y confeti.

📱 Diseño responsive

Las orlas se adaptan automáticamente a cualquier tamaño de pantalla: móviles, tablets y escritorio. Las fichas, fotos, textos y márgenes se reajustan mediante unidades cqw (container query width) para verse proporcionadas en cualquier dispositivo.

📐 Distribución inteligente de alumnos

El sistema distribuye los alumnos en filas usando flex-wrap y ajusta automáticamente el tamaño según la cantidad:

AlumnosAncho por alumnoFotoGap vertical
Menos de 1211 cqw6.5 cqw1 cqw
12 a 2010 cqw5.5 cqw0.6 cqw
21 a 258.5 cqw4.8 cqw0.5 cqw
Más de 257.5 cqw4 cqw0.4 cqw

El sistema evita que la última fila se quede con un único alumno (efecto huérfano), ajustando márgenes para que la composición sea siempre simétrica y equilibrada.

🔥 Normalizador de Fotos

El proyecto incluye normalizador-fotos.html, una herramienta visual que prepara automáticamente las fotografías para las orlas: las recorta en cuadrado, centra el rostro y las unifica al mismo tamaño.

🤔 ¿Por qué es necesario?

En una orla todas las fotos deben verse uniformes: mismo tamaño, misma composición, rostro centrado. Con fotos de diferentes fuentes (móviles, cámaras, redes sociales) es difícil lograr consistencia manualmente. Esta herramienta lo hace automático en lote.

🧠 Cómo funciona internamente

  1. Carga masiva — Arrastra todas las fotos que quieras (JPG, PNG, WebP) o selecciónalas con el selector de archivos.
  2. Detección facial — Usa la API nativa FaceDetector de Chromium (Chrome, Edge, Brave, Opera) para localizar el rostro. No necesita librerías externas ni conexión a internet.
  3. Recorte inteligente — Calcula el encuadre óptimo alrededor del rostro con un margen configurable. Si el rostro está cerca del borde, desplaza el encuadre para no recortar fuera de la imagen.
  4. Redimensionado — Escala el recorte al tamaño de salida (600×600 px por defecto) con la máxima calidad de imagen.
  5. Previsualización — Muestra cada foto Original vs Normalizada lado a lado en tarjetas.
  6. Descarga — Individual o todas juntas (usando File System Access API si está disponible, o descarga secuencial).

🐍 Normalizador con Python + OpenCV

normalizar-fotos.py es una versión por línea de comandos que usa OpenCV para detectar rostros con dos motores:

MotorPrecisiónVelocidadDescripción
Haar Cascade⚠️ Básica🚀 RápidoIntegrado en OpenCV, sin descargas adicionales
DNN (SSD+ResNet)✅ Alta⚡ MedioSe descarga automáticamente (~20 MB) la primera vez

Instalación

pip install -r requirements.txt

Uso directo

python normalizar-fotos.py
python normalizar-fotos.py --input ./profesores --size 800 --detector dnn
python normalizar-fotos.py --input ./alumnos --format png --verbose

Lanzador automático (Windows)

ejecutar-normalizador.ps1 hace todo solo: crea un entorno virtual, instala dependencias, descarga el modelo DNN y ejecuta el normalizador con los argumentos que le pases:

.\ejecutar-normalizador.ps1
.\ejecutar-normalizador.ps1 -Input ./profesores -Detector dnn -Verbose
.\ejecutar-normalizador.ps1 -DryRun

🖥️ Normalizador con ImageMagick

normalizar-fotos.ps1 usa ImageMagick para centrado geométrico y redimensionado en lote. No tiene detección facial, pero es la opción más rápida si todas las fotos ya están bien encuadradas.

.\normalizar-fotos.ps1
.\normalizar-fotos.ps1 -InputDir ./profesores -Size 800 -Verbose
.\normalizar-fotos.ps1 -InputDir ./fotos -DryRun

Requiere ImageMagick instalado y disponible en el PATH.

📝 Guía completa: Cómo crear tu propia orla

Sigue estos pasos para personalizar una orla con tus propios datos y fotografías:

Paso 1: Elige una plantilla base

Parte de cualquiera de los archivos HTML existentes. El más recomendado es index-ejemplo-principal.html (plantilla principal que contiene el diseño base). Si tu ciclo tiene un número de alumnos similar a algún ejemplo, usa ese como referencia:

📖 Hasta 12 → Asistencia a la Dirección 💻 15–20 → SMR (17 alumnos) 📱 21–25 → DAM (25 alumnos) 📚 26–30 → 4º ESO / Bachillerato

Paso 2: Normaliza las fotografías

Tienes tres opciones para dejar todas las fotos con el mismo tamaño y la cara centrada:

🌐
HTML (navegador)
Abre normalizador-fotos.html en Chrome/Edge, arrastra las fotos y pulsa "Descargar ZIP".
🐍
Python + OpenCV
.\ejecutar-normalizador.ps1 -Detector dnn — instala y ejecuta todo.
🖥️
ImageMagick
.\normalizar-fotos.ps1 — centrado geométrico rápido.

Coloca las fotos resultantes en las carpetas:

  • profesores/nombre_apellido1_apellido2.jpg — para el profesorado
  • alumnos/nombre_apellido1_apellido2.jpg — para el alumnado

Si una foto no existe, el sistema muestra automáticamente un círculo con la inicial de la persona.

Paso 3: Edita los datos

Busca en el HTML los arrays profesoresRaw y alumnosRaw y edítalos:

const profesoresRaw = [
  { nombre: 'María García López', cargo: 'Tutora' },
  { nombre: 'Juan Pérez Martínez', cargo: 'Profesor de Matemáticas' }
];

const alumnosRaw = [
  { nombre: 'Ana',   apellidos: 'Gómez Ruiz' },
  { nombre: 'Luis',  apellidos: 'Sánchez López' }
];

Paso 4: Personaliza el diseño

Modifica las variables CSS en el bloque :root para ajustar los colores:

:root {
  --bg-body: #f3f0e9;       /* Fondo general */
  --bg-orla: #fefcf5;       /* Fondo de la orla */
  --accent: #2c482b;        /* Color principal */
  --accent-alt: #8b6239;    /* Color secundario */
  --text-main: #2c2c2c;
  --text-light: #6b5a45;
}

Reemplaza layout.png por tu propia imagen de fondo (1920×1080 px recomendado).

Paso 5: Actualiza el texto

<div class="ciclo">CICLO FORMATIVO DE GRADO MEDIO</div>
<div class="promocion">PROMOCIÓN 2024-2026</div>

Paso 6: Abre en el navegador

No necesitas servidor. Abre el archivo .html directamente con doble clic en Chrome o Edge. Todo funciona 100% desde el sistema de archivos local.

🤖 Scripts de automatización

El proyecto incluye scripts en PowerShell para automatizar la generación de orlas de ejemplo:

🔨
update_orlas.ps1
Genera todas las orlas desde cero: nombres ficticios, asignación de fotos y HTML completos.
🔄
sync_all.ps1
Sincroniza el diseño de todas las orlas desde la plantilla index-ejemplo-principal.html.
🔍
check_missing.ps1
Verifica qué fotografías faltan en los directorios.
📋
copy_photos.ps1
Distribuye fotos desde el pool photos/ a los directorios normalizados.
🐍
ejecutar-normalizador.ps1
Lanzador todo-en-uno: crea venv, instala dependencias, descarga modelo DNN y ejecuta normalizar-fotos.py.
📦
requirements.txt
Dependencias Python para normalizar-fotos.py. Instalar con pip install -r requirements.txt.

Los scripts usan un pool de fotografías genéricas (photos/a1.jpg … p6.jpg) y nombres ficticios para producir orlas completas de ejemplo. Son ideales para hacer una demo rápida o probar cambios en el diseño.

🌍 Publicar en GitHub Pages

Para compartir tus orlas en internet:

  1. Sube el proyecto a GitHub — crea un repositorio y sube todos los archivos.
  2. Activa GitHub Pages — ve a Settings → Pages, selecciona la rama main y la carpeta /docs.
  3. En unos segundos tu sitio estará disponible en https://tu-usuario.github.io/GeneradorOrlas/.

El contenido de la carpeta docs/ se sirve automáticamente como sitio web estático, sin necesidad de configuración adicional.

⚠️ Aviso importante

Todos los nombres, apellidos y datos personales que aparecen en las orlas de ejemplo son completamente ficticios. Han sido generados algorítmicamente combinando nombres y apellidos comunes de forma aleatoria.

Todas las fotografías son igualmente ficticias: imágenes genéricas de archivo o placeholders. Cualquier parecido con la realidad es mera coincidencia.

Este proyecto se proporciona como herramienta de ejemplo y plantilla para que cada centro educativo pueda adaptarlo con sus propios datos oficiales.
